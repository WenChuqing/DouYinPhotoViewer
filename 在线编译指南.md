## 📖 平台一：GitHub Actions 使用教程

### 1. 上传项目到 GitHub

**步骤截图描述：**
1. 登录 GitHub，点击右上角 `+` → `New repository`
2. 填写仓库名称（如：`DouYinPhotoViewer`）
3. 选择 Public 或 Private，点击 `Create repository`
4. 在本地项目目录执行以下命令：

```bash
cd 抖音看图
git init
git add -A
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```

### 2. 触发构建

**步骤截图描述：**
1. 进入 GitHub 仓库页面
2. 点击顶部的 `Actions` 标签
3. 左侧选择 `Build Android APK` 工作流
4. 点击右侧的 `Run workflow` 按钮
5. 选择分支（main），再次点击 `Run workflow`

### 3. 查看构建状态

**步骤截图描述：**
1. 在 Actions 页面可以看到正在运行的构建任务
2. 点击构建任务可以查看实时日志
3. 构建通常需要 3-8 分钟完成
4. 绿色对勾表示构建成功，红色叉号表示失败

### 4. 下载 APK 文件

**步骤截图描述：**
1. 点击成功的构建记录
2. 滚动到页面底部的 `Artifacts` 区域
3. 可以看到两个文件：
   - `DouYinPhotoViewer-debug` - Debug 版本 APK
   - `DouYinPhotoViewer-release` - Release 版本 APK
4. 点击文件名即可下载 ZIP 压缩包
5. 解压后得到 APK 文件

## 📖 平台二：Bitrise 使用教程

### 1. 注册并添加项目

**步骤截图描述：**
1. 访问 https://bitrise.io 注册账号
2. 登录后点击 `+ Add new app`
3. 选择连接 GitHub / GitLab / Bitbucket
4. 选择您的项目仓库
5. 选择分支（main 或 master）
6. Bitrise 会自动检测项目类型（Android）

### 2. 配置构建流程

**步骤截图描述：**
1. 检测完成后，确认项目类型为 Android
2. 选择构建变体：`debug` 或 `release`
3. 在 `Workflow Editor` 中检查构建步骤
4. 确保包含以下步骤：
   - `Git Clone Repository`
   - `Install missing Android SDK components`
   - `Android Build`

### 3. 开始构建

**步骤截图描述：**
1. 点击 `Start/Schedule a Build` 按钮
2. 确认配置后点击 `Start Build`
3. 等待构建完成（5-10 分钟）
4. 构建日志实时显示在页面上

### 4. 下载 APK

**步骤截图描述：**
1. 构建成功后，在 `Apps & Artifacts` 标签页
2. 找到生成的 APK 文件
3. 点击下载图标保存到本地

## 📖 平台三：GitLab CI/CD 使用教程

### 1. 上传项目到 GitLab

**步骤截图描述：**
1. 登录 GitLab，点击 `New project`
2. 选择 `Create blank project`
3. 填写项目名称，点击 `Create project`
4. 按照页面提示将代码推送到 GitLab

### 2. 创建 CI 配置文件

在项目根目录创建 `.gitlab-ci.yml` 文件：

```yaml
image: eclipse-temurin:17-jdk

variables:
  ANDROID_COMPILE_SDK: "34"
  ANDROID_BUILD_TOOLS: "34.0.0"
  ANDROID_SDK_TOOLS: "11076708"

before_script:
  - apt-get --quiet update --yes
  - apt-get --quiet install --yes wget tar unzip lib32stdc++6 lib32z1
  - wget --quiet --output-document=android-sdk.zip https://dl.google.com/android/repository/commandlinetools-linux-${ANDROID_SDK_TOOLS}_latest.zip
  - unzip -d android-sdk-linux android-sdk.zip
  - echo y | android-sdk-linux/cmdline-tools/bin/sdkmanager --sdk_root=android-sdk-linux "platforms;android-${ANDROID_COMPILE_SDK}" >/dev/null
  - echo y | android-sdk-linux/cmdline-tools/bin/sdkmanager --sdk_root=android-sdk-linux "platform-tools" >/dev/null
  - echo y | android-sdk-linux/cmdline-tools/bin/sdkmanager --sdk_root=android-sdk-linux "build-tools;${ANDROID_BUILD_TOOLS}" >/dev/null
  - export ANDROID_HOME=$PWD/android-sdk-linux
  - export PATH=$PATH:$PWD/android-sdk-linux/platform-tools/
  - chmod +x ./gradlew

stages:
  - build

build:
  stage: build
  script:
    - ./gradlew assembleDebug
  artifacts:
    paths:
      - app/build/outputs/apk/debug/*.apk
  cache:
    key: "$CI_COMMIT_REF_NAME"
    paths:
      - .gradle
```

### 3. 触发构建并下载

**步骤截图描述：**
1. 提交并推送 `.gitlab-ci.yml` 文件
2. GitLab 会自动触发构建
3. 点击左侧 `CI/CD` → `Jobs` 查看状态
4. 构建成功后，点击 Job 详情页面的 `Download` 按钮
5. 或在 `CI/CD` → `Artifacts` 页面下载

## 🔧 常见问题和排错指南

### 问题 1：构建失败，提示内存不足

**错误信息：**
```
java.lang.OutOfMemoryError: Java heap space
```

**解决方案：**
1. 检查 `gradle.properties` 中的 `org.gradle.jvmargs` 配置
2. 确保已设置 `-Xmx2048m` 或更大
3. 在 CI 环境变量中增加：`GRADLE_OPTS=-Xmx2048m`

---

### 问题 2：Gradle 下载依赖超时

**错误信息：**
```
Could not resolve com.android.tools.build:gradle:8.2.0
```

**解决方案：**
1. 检查网络连接
2. 在 `build.gradle` 中确认使用了国内镜像源
3. 重试构建（通常是临时网络问题）

---

### 问题 3：SDK 版本不兼容

**错误信息：**
```
SDK location not found or compileSdkVersion is not specified
```

**解决方案：**
1. 确认 `app/build.gradle` 中的 `compileSdk` 设置为 34
2. 在线平台通常会自动安装所需 SDK 版本
3. 手动在平台配置中添加 SDK 安装步骤

---

### 问题 4：gradlew 权限问题

**错误信息：**
```
Permission denied: ./gradlew
```

**解决方案：**
1. 在构建脚本中添加权限设置：
   ```bash
   chmod +x ./gradlew
   ```
2. Git 中保留执行权限：
   ```bash
   git update-index --chmod=+x gradlew
   ```

---

### 问题 5：Release 构建失败（签名问题）

**错误信息：**
```
Keystore file not found for signing config 'release'
```

**解决方案：**
1. Debug 版本可以正常构建，无需签名
2. 如果需要 Release 签名：
   - 在平台的 Secrets/Environment Variables 中配置签名信息
   - 或使用平台提供的自动签名功能
   - 本项目默认可以构建无签名的 Release APK

---

### 问题 6：构建时间过长

**优化建议：**
1. 启用 Gradle 缓存（已在 gradle.properties 中配置）
2. 在 CI 配置中启用缓存：
   - 缓存 `.gradle` 目录
   - 缓存 `~/.gradle/caches`
3. 使用并行构建（已配置 `org.gradle.parallel=true`）

## 📱 编译成功后如何安装到手机

### 方法一：直接传输安装

**步骤截图描述：**
1. 将下载的 APK 文件通过 USB 数据线传到手机
2. 或通过微信/QQ 文件助手发送到手机
3. 在手机文件管理器中找到 APK 文件
4. 点击 APK 文件开始安装

### 方法二：ADB 安装（开发者推荐）

**前置条件：**
- 手机已开启 USB 调试
- 电脑已安装 ADB 工具

**安装命令：**
```bash
# 连接设备
adb devices

# 安装 APK
adb install -r app-debug.apk
```

### 方法三：二维码安装

**步骤截图描述：**
1. 将 APK 上传到网盘（如百度网盘、蓝奏云）
2. 生成下载链接的二维码
3. 用手机扫描二维码下载并安装

### 安装时的注意事项

⚠️ **安全提示：**
1. 首次安装时，系统会提示"未知来源应用"
2. 在设置中允许安装来自该来源的应用
3. 安装完成后建议关闭此权限

🔍 **安装验证：**
1. 安装后在应用列表中找到"抖音看图"
2. 点击打开确认应用正常运行
3. 检查主要功能是否正常工作

## 💡 进阶技巧

### 1. 自动签名配置

如需配置自动签名，在 `app/build.gradle` 中添加：

```gradle
android {
    signingConfigs {
        release {
            storeFile file(System.getenv("KEYSTORE_FILE"))
            storePassword System.getenv("KEYSTORE_PASSWORD")
            keyAlias System.getenv("KEY_ALIAS")
            keyPassword System.getenv("KEY_PASSWORD")
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
        }
    }
}
```

然后在平台的环境变量中配置对应的密钥信息。

### 2. 构建缓存配置

在 GitHub Actions 中添加 Gradle 缓存：

```yaml
- name: Cache Gradle packages
  uses: actions/cache@v4
  with:
    path: |
      ~/.gradle/caches
      ~/.gradle/wrapper
    key: ${{ runner.os }}-gradle-${{ hashFiles('**/*.gradle*', '**/gradle-wrapper.properties') }}
    restore-keys: |
      ${{ runner.os }}-gradle-
```

### 3. 多渠道构建

如需构建多个渠道版本，可以在 GitHub Actions 中配置矩阵构建：

```yaml
strategy:
  matrix:
    flavor: [free, pro]
```
